# 🏛️ دستور مشروع صُنّاع السعادة (Saade Makers Constitution)

## 📌 المبادئ الأساسية (لا يجوز الإخلال بها تحت أي ظرف)

### 1. نظام الأدوار والصلاحيات
| الدور | الوصف | الصلاحيات الأساسية |
|-------|-------|--------------------|
| `owner` | مالك المتجر | جميع الصلاحيات بلا استثناء |
| `manager` | مدير فرع | إدارة المنتجات، الطلبات، الأقسام، التصدير |
| `employee` | موظف | عرض المنتجات، إدارة الطلبات، الاستيراد |
| `customer` | عميل | لا صلاحيات إدارية |

### 2. قواعد المزامنة مع Nard POS (لا تُخالف أبداً)
- ✅ المطابقة بالباركود فقط (barcode column أو description field)
- ✅ منع التكرار تحت أي حال من الأحوال
- ✅ لو المتجر رصيده صفر لصنف ونفس الباركود في السيستم له رصيد → يتم تحديث الرصيد
- ✅ الأولوية للصنف اللي له صورة عند وجود تكرار
- ✅ الأسعار من السيستم تشمل الضريبة مسبقاً (لا تُضاف ضريبة إضافية)
- ✅ المبيعات = الفرق بين آخر مزامنتين فقط (غير تراكمية)
- ✅ القيم تُحفظ في localStorage وتبقى حتى المزامنة التالية

### 3. قواعد البحث والفلترة
- ✅ البحث يعمل داخل الفلتر النشط (الزر المحدد)
- ✅ استخدام fetchIdRef لمنع Race Conditions
- ✅ البحث بيبحث في الاسم والوصف (يشمل الباركود)

### 4. قواعد تصدير الإكسيل
- ✅ لو في منتجات محددة (checkboxes) → يصدر المحدد فقط
- ✅ لو مافيش تحديد → يصدر كل المنتجات الظاهرة حسب الفلتر النشط
- ✅ الملف يتسمى باسم الفلتر النشط + التاريخ

### 5. قواعد حماية الجلسة (Constitution Hook)
- ✅ الدور يتخزن في sessionStorage عند تسجيل الدخول
- ✅ لو تم تعديل الدور يدوياً → الصفحة تتحمل من جديد
- ✅ مراقبة cross-tab storage changes

## 📁 هيكل ملفات النظام

```
src/
├── auth/
│   ├── roles.ts          # Re-exports from use-auth.ts
│   └── permissions.ts    # Re-exports from use-auth.ts
├── hooks/
│   ├── use-auth.ts       # 🏛️ المصدر الرئيسي للأدوار والصلاحيات
│   ├── use-constitution.ts  # 🏛️ حارس الدستور
│   └── use-admin-dashboard.ts  # منطق لوحة التحكم
├── components/admin/
│   ├── AdminStats.tsx    # بطاقات الإحصائيات
│   ├── AdminTabs.tsx     # أزرار التبويب والبحث والتصدير
│   ├── ProductTable.tsx  # جدول المنتجات
│   └── UserManagement.tsx # إدارة المستخدمين
└── App.tsx               # يستدعي useConstitution()
```

## ⚠️ تعليمات للمطورين

> **قبل أي تعديل في الكود:**
> 1. اقرأ هذا الملف كاملاً
> 2. تأكد أن تعديلك لا يخالف أي قاعدة مذكورة أعلاه
> 3. لا تنسخ تعريف الأدوار في ملف جديد - استخدم `use-auth.ts` فقط
> 4. لا تعدل `hasPermission` إلا من `use-auth.ts`
> 5. لا تمسح localStorage keys المحمية (whitelist in App.tsx)
